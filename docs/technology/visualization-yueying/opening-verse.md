# 开篇词

## 前端工程师为什么要学可视化？

1. 现在很多 C 端或 B 端的产品都离不开可视化
   - 淘宝可视化大屏
   - 企业级应用中的态势感知和指挥调度大屏
   - 智慧城市、智慧社区、智慧生活
2. 可视化能实现传统 Web 无法实现的效果
3. 能力提升，可以深入掌握图形学和 WebGL 技术，拓宽技术成长路线

## 如何成为一名可视化工程师？

一名优秀的可视化工程师，最应当具备的能力就是要根据产品经理和设计师的想法，实现真正贴合用户需求的、灵活多变的、具有视觉震撼力，并且充满创意的优秀可视化作品。

想要成为一名可视化工程师，我们不能停留在可视化现有的库、框架和工具的使用层面上，必须深入技术栈的底层，充分理解数据，掌握视觉呈现相关的技术和基本原理。打好基础之后，再系统地学习可视化领域的方法论，才能提高解决问题的能力。

## 什么事可视化

可视化是将数据组织成易于为人所理解和认知的结构，然后用图形的方式形象地呈现出来的理论、方法和技术。

实现可视化有两个关键要素：数据和图形。如果要在计算机上呈现，还要加上交互。

### Web 前端和可视化区别

1. 技术栈不同。Web 开发主要以 HTML 来描述结构，以 CSS 来描述表现，以 JavaScript 来描述行为。可视化较少涉及 HTML 和 CSS，主要同 Canvaas、SVG、WebGL 等图形 API 打交道。
2. Web 开发着重于处理普通的和文本和多媒体信息，可视化开发着重于处理结构化数据，渲染各种相对复杂的图表和图形元素。
